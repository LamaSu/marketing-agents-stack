# @mstack/cli (`mstack`)

The offline demo driver that ties the whole stack into one runnable **signal → decision → action** loop. `mstack seed && mstack demo` runs **end-to-end offline with no credentials** and exits 0: it seeds the bundled sample signals + accounts + north-star corpus into the DuckDB warehouse (and the LanceDB reviewer corpus), then runs BOTH production workflows through `@mstack/runtime` — `content-review` over the sample partner assets (deterministic claim-drift review → RETURNED/APPROVED + partner-email/export drafts) and `account-activation` over the sample signals (context-engine + `RulesScorer` → a decision + a templated outreach draft). Every candidate action lands under `drafts/` as `pending` and **nothing is dispatched** — the demo asserts an empty outbox on the way out (guardrail #2: a human approves every send). Mode is **live** iff `ANTHROPIC_API_KEY` is set (Claude for extraction/judgment/copy via `reviewAsset` + the account-intel swarm), else **offline** (the deterministic + rules + fixture path); the same subcommands drive both. Other commands: `mstack list` (pending drafts), `mstack approve <draftId>` (the one gated path to an outbox send, hash-chain verified), `mstack review <file>`, `mstack score <domain>`. Do not run `pnpm install`/build locally (the dev tablet OOMs on the native deps) — Spark verifies.
